## 🚀 How to Use This Repository

```bash
# 📥 1. Clone the repository
git clone https://github.com/YOUR_USERNAME/dsa_sheet.git

# 📂 2. Navigate into the folder
cd "DSA SHEET"

# 🗂️ 3. Open a specific step/topic
cd "Step 03 — Arrays"

# ⚙️ 4. Compile any solution
g++ -std=c++17 -O2 -Wall -o solution ProblemName.cpp

# ▶️ 5. Run it
./solution
```

<br/>

<div align="center">

| Command | Purpose |
|:--------|:--------|
| `g++ -std=c++17 -O2 -o sol file.cpp` | Compile with C++17 & optimization |
| `g++ -std=c++17 -Wall -Wextra -o sol file.cpp` | Compile with full warnings (dev mode) |
| `./sol` | Run the compiled output |
| `valgrind ./sol` | Check for memory leaks |

</div>

> 💡 **Tip:** Recommended to use **VS Code** with the **C/C++ extension** for the best experience.  
> Add `"C_Cpp.default.cppStandard": "c++17"` in your `settings.json`.

## 🤝 Contributing

<div align="center">

```
Contributions are what make the open-source community truly amazing. 🌟
Whether it's a better approach, a bug fix, or a missing solution — you're welcome here!
```

</div>

### 📌 How to Contribute — Step by Step

**Step 1 —** 🍴 **Fork** the repository by clicking the **Fork** button at the top right.

**Step 2 —** 📥 **Clone** your forked copy locally:
```bash
git clone https://github.com/YOUR_FORK_USERNAME/dsa_sheet.git
cd dsa_sheet
```

**Step 3 —** 🌿 **Create a new branch** (use a descriptive name):
```bash
# Convention: feat/add-[problem-name]-[topic]
git checkout -b feat/add-two-sum-arrays
```

**Step 4 —** ✍️ **Write your solution** using the file template above, placed in the correct folder.

**Step 5 —** 💾 **Stage and commit** with a meaningful message:
```bash
git add .
git commit -m "✨ feat: Add Two Sum — optimal HashMap approach (Arrays)"
```

**Step 6 —** 🚀 **Push** to your fork:
```bash
git push origin feat/add-two-sum-arrays
```

**Step 7 —** 🔁 **Open a Pull Request** from your branch to `main` of this repo.

<br/>

### ✅ Contribution Checklist

Before submitting a PR, make sure:

- [x] Solution is placed in the **correct Step/Topic folder**
- [x] File follows the **solution template** (problem link, complexity, approach)
- [x] Code is **clean, commented** and compiles without errors in C++17
- [x] Includes **time & space complexity** in the header comment
- [x] Commit message uses the **emoji + convention** format
- [x] PR description clearly states what problem is being added/fixed
- [ ] ❌ Do **not** add non-working, incomplete, or plagiarized solutions
- [ ] ❌ Do **not** modify the folder structure without first opening an Issue

<br/>

### 🏷️ Commit Message Convention

```
✨ feat   → New solution added
🐛 fix    → Bug fix in existing solution
♻️  refactor → Better approach to existing solution
📝 docs   → README or comment improvements
🗂️  chore  → File organization, renaming
```
